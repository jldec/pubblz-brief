Opportunities for GitHub Pages

Github Pages is awesome!

  • Zero-config CDN hosting
  • Great http performance, fast invalidation
  • Compression, expiration, mime-types just work
  • Minimal-config own CNAME
  • Git publishing workflow
  • Free!

With some limitations

  • no HTTPS
  • no form handling
  • no private pages
  • no repo authz
  • Jekyll-only SSG
  • no proxy routes
  • no CMS / editor

Github Pages users today

  • Have to be ok with git, command line, text editor
  • Work on a machine running OSX/Linux/Windows
  • Manage files in a file system
  • Jeykyll users also have to be ok with Ruby

What is pub-server?

  • Static Site Generator (like Jekyll)
  • Database-free Markdown-driven CMS
  • Browser-based editor
  • 100% JavaScript
  • Extensible via npm modules

Web publishing

200M users on Automattic + SquareSpace + Wix + Weebly

Mobile & Responsive & ES6

  • Driving a sea-change in web publishing
  • Favoring frameworks like React (and pub-server)
  • Written in Javascript
  • Entirely new ecosystem of tooling and components
  • Designed to render BOTH in-browser and on-server
  • Developers don't want to write templates and helpers twice

Front-end components

This is a window of opportunity for GitHub to offer Web developers better tools.

Front-end component CDN hosting

  • Record and publish download counts
  • Performance and error logging
  • Analytics for developers

Front-end component registry

  • npm for Web devs
  • Searchable by downloads
  • Search by browser/framework compatibility

GitHub Pages for documentation

  • Combination gh-pages and wiki
  • Source repo: https://github.com/Netflix/Hystrix.git
  • Pages repo: https://github.com/Netflix/Hystrix.pages.git
  • Auto-published just like gh-pages
  • Also support CNAMEs
  • or https://github/com/Netflix/Hystrix/pages/ (not sure about CNAME)
  • Pub-server would enable custom themes and wysiwyg editing
